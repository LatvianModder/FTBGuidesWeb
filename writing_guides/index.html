Formatting

Structure

Each guide folder looks like this:

data.json
index.html
subpage/
	data.json
	index.html
	subpage/
		data.json
		index.html
		...
subpageNoText/
	data.json
	subpage1/
		data.json
		index.html
	subpage2/
		data.json
		index.html

index.html

All guide files are written in html, but they don't require html, head or body tags. You must treat each file, as if it's already inside body tag. Other than that, it's a normal html file.

Guides look for index.html files inside folders, all other files will be ignored.

data.json

data.json file must be added in each directoy where index.html exists.

Key Type Required Description
title String true Lowercase ID
type String true* Type of the guide, e.g. "mod". Used for sorting / easier viewing.
icon String false Absolute path to icon. You can use items as icons, e.g. "item:minecraft:apple". If this key is missing, icon.png from current folder will be used.
authors String[] false* List of mod/modpack authors. If you only made the guide, don't list yourself here.
subpages String[] false List of subpage IDs. If this key is missing or value is empty, no subpages will be listed.

*Only used in the main data.json.

Supported HTML tags

All tags are supported in web browser, but these are supported by the mod:

  • p - Paragraph
  • a - Links & Formatting
  • b, i, u, s - Bold, Italic, Underlined, Striketrough
  • img - Image
  • h1, h2, h3 - Headings
  • ul, ol, li - Lists
  • pre, code - Code
  • br - Line Break
  • hr - Horizontal Line
  • table, tr, th, td - Tables

Special tags & classes

  • .mc - Class that hides tags in browser.
  • .nomc - Class that hides tags in Minecraft. Add it to non-supported tags.
  • recipe

FTBGuides mod & Special Guides

With the FTBGuides mod installed, you can:

  • View all guides ingame by clicking Guides button in inventory and save them for offline use.
  • Add a modpack-specific guide in config/ftbguides/modpack folder.
  • Add a server-specific guide in local/ftbguides/server folder.
